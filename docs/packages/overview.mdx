## Package Categories

### Core Development

<AccordionGroup>
  <Accordion title="EVM Development (@sei-js/evm)">
    **Best for**: Developers familiar with Ethereum ecosystem
    
    **Features**:
    - Wagmi/Viem integration
    - Ethers.js support
    - Pre-configured Sei network chains
    - React hooks for wallet connection
    - TypeScript-first development
    
    **Use cases**:
    - DeFi applications
    - NFT marketplaces
    - Token swaps
    - Multi-chain applications
  </Accordion>

  <Accordion title="Cosmos Development (@sei-js/cosmjs)">
    **Best for**: Developers building on Cosmos ecosystem
    
    **Features**:
    - CosmJS integration
    - Wallet connection utilities
    - Transaction signing
    - RPC querying
    - Address utilities
    
    **Use cases**:
    - CosmWasm smart contracts
    - IBC transfers
    - Staking applications
    - Governance tools
  </Accordion>

  <Accordion title="Protocol Integration (@sei-js/cosmos)">
    **Best for**: Low-level protocol integration
    
    **Features**:
    - Protobuf encoding/decoding
    - REST query client
    - TypeScript types for Sei Cosmos
    - Message construction utilities
    
    **Use cases**:
    - Custom transaction builders
    - Protocol explorers
    - Advanced integrations
  </Accordion>
</AccordionGroup>

### Developer Tools

<AccordionGroup>
  <Accordion title="Project Scaffolding (@sei-js/create-sei)">
    **Best for**: Starting new projects quickly
    
    **Features**:
    - Interactive CLI wizard
    - Multiple framework support (Next.js, Vite)
    - Pre-configured templates
    - Both EVM and Cosmos templates
    
    **Templates**:
    - Next.js + Wagmi (EVM)
    - Next.js + Ethers (EVM)
    - Vite + CosmJS (Cosmos)
    - Custom configurations
  </Accordion>

  <Accordion title="Hardware Wallets (@sei-js/ledger)">
    **Best for**: Enterprise and security-focused applications
    
    **Features**:
    - Ledger device integration
    - Secure transaction signing
    - Multi-signature support
    - Hardware-backed key management
    
    **Use cases**:
    - Enterprise applications
    - High-value transactions
    - Institutional tools
  </Accordion>
</AccordionGroup>

### Wallet & Infrastructure

<AccordionGroup>
  <Accordion title="Global Wallet (@sei-js/sei-global-wallet)">

    **Features**:
    - EIP-6963 compliance
    - Account abstraction support
    - Cross-dApp compatibility
    - Unified wallet interface
    
    **Benefits**:

    - Reduced wallet switching

  <Accordion title="Chain Registry (@sei-js/registry)">

    - RPC endpoint management

    **Networks**:
    - Sei Mainnet (Pacific-1)
    - Sei Testnet (Atlantic-2)
    - Sei Devnet
  </Accordion>
</AccordionGroup>

## Installation Guide

### Complete Installation

For full SeiJS functionality:

```bash
npm install @sei-js/evm @sei-js/cosmjs @sei-js/cosmos
```

### EVM-Only Setup

For EVM-focused development:

```bash
npm install @sei-js/evm ethers viem wagmi
```

### Cosmos-Only Setup

For Cosmos-focused development:

```bash
npm install @sei-js/cosmjs @sei-js/cosmos
```

### Development Tools

For project scaffolding and development:

```bash
npm install -g @sei-js/create-sei
```

## Migration Guide

### From v1.x to v2.x

<Steps>
  <Step title="Update Dependencies">
    Update all @sei-js packages to v2.x
    ```bash
    npm update @sei-js/*
    ```
  </Step>
  
  <Step title="Update Imports">
    Some import paths have changed:
    ```typescript
    // Old
    import { getSigningClient } from '@sei-js/cosmjs/lib/client';
    
    // New
    import { getSigningClient } from '@sei-js/cosmjs';
    ```
  </Step>
  
  <Step title="Update Configuration">
    Network configurations have been updated:
    ```typescript
    // Old
    import { MAINNET_CHAIN } from '@sei-js/evm/chains';
    
    // New
    import { SEI_MAINNET } from '@sei-js/evm';
    ```
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Package Selection">
    - Use `@sei-js/evm` for Ethereum-style development
    - Use `@sei-js/cosmjs` for Cosmos-native features
    - Combine packages as needed for hybrid applications
    - Start with `@sei-js/create-sei` for new projects
  </Accordion>

  <Accordion title="Performance">
    - Import only what you need to reduce bundle size
    - Use tree-shaking compatible bundlers
    - Cache client instances when possible
    - Implement proper error handling
  </Accordion>

  <Accordion title="Security">
    - Never expose private keys in client-side code
    - Use hardware wallets for high-value operations
    - Validate all user inputs
    - Implement proper access controls
  </Accordion>
</AccordionGroup>
