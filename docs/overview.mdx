## Prerequisites

Before you start building, ensure you have the following installed:

<AccordionGroup>
  <Accordion title="Node.js & Package Manager" icon="node-js">
    **Required versions:**
    - Node.js v18 or higher
    - npm, yarn, or pnpm (latest version)
    
    <Tabs>
      <Tab title="Check Node.js">
        ```bash
        node --version
        # Should output v18.0.0 or higher
        ```
      </Tab>
      <Tab title="Install Node.js">
        Download from [nodejs.org](https://nodejs.org) or use a version manager:
        ```bash
        # Using nvm
        nvm install --lts
        nvm use --lts
        ```
      </Tab>
    </Tabs>
  </Accordion>

  <Accordion title="Code Editor (Recommended)" icon="code">
    **VS Code with TypeScript support**
    
    We recommend [Visual Studio Code](https://code.visualstudio.com/) with the following extensions:
    - TypeScript and JavaScript Language Features (built-in)
    - Solidity extension (for smart contracts)
    - Tailwind CSS IntelliSense (for styling)
  </Accordion>
</AccordionGroup>

## Create your first Sei application

<Steps>
  <Step title="Initialize your project">
    Use our CLI tool to create a new Sei application with everything pre-configured:
    
    <CodeGroup>
      ```bash npm
      npx @sei-js/create-sei my-sei-app
      ```
      
      ```bash yarn
      yarn create @sei-js/create-sei my-sei-app
      ```
      
      ```bash pnpm
      pnpm create @sei-js/create-sei my-sei-app
      ```
    </CodeGroup>
    
    <Check>
    **Expected outcome:** A new project directory with TypeScript configuration, testing setup, and example code.
    </Check>
    
    <Info>
    The CLI will ask you to choose a template. Select "React + Wagmi" for a full-featured web application, or "Node.js" for backend/script development.
    </Info>
  </Step>

  <Step title="Navigate and install dependencies">
    Move into your project directory and install dependencies:
    
    ```bash
    cd my-sei-app
    npm install
    ```
    
    <Check>
    **Expected outcome:** All dependencies installed successfully without errors.
    </Check>
    
    <Accordion title="Troubleshooting installation issues">
      **Common issues and solutions:**
      
      - **Node version conflicts:** Use `nvm use` to switch to the correct Node.js version
      - **Permission errors:** Avoid using `sudo` with npm. Use `nvm` or fix npm permissions
      - **Network timeouts:** Try switching to a different registry: `npm config set registry https://registry.npmjs.org/`
    </Accordion>
  </Step>

  <Step title="Configure your environment">
    Create a `.env` file in your project root with your Sei network configuration:
    
    ```bash .env
    # Network Configuration
    NEXT_PUBLIC_CHAIN_ID=sei
    NEXT_PUBLIC_RPC_URL=https://rpc.sei-apis.com
    NEXT_PUBLIC_REST_URL=https://rest.sei-apis.com
    
    # Optional: For development
    NEXT_PUBLIC_ENABLE_TESTNETS=true
    ```
    
    <Warning>
    Never commit private keys or sensitive information to version control. Use environment variables for all secrets.
    </Warning>
    
    <Check>
    **Expected outcome:** Environment file created with network endpoints configured.
    </Check>
  </Step>

  <Step title="Start your development server">
    Launch your application in development mode:
    
    ```bash
    npm run dev
    ```
    
    <Check>
    **Expected outcome:** Development server running at `http://localhost:3000` with hot reload enabled.
    </Check>
    
    Your browser should automatically open to show your new Sei application with:
    - Wallet connection button
    - Network status display
    - Example interactions ready to test
  </Step>
</Steps>

## Test your application

<Tabs>
  <Tab title="Connect a Wallet">
    **Test wallet connectivity:**
    
    1. Click "Connect Wallet" in your application
    2. Select a wallet (MetaMask, Compass, or any Sei-compatible wallet)
    3. Approve the connection request
    
    <Check>
    You should see your wallet address displayed and network status showing "Connected to Sei".
    </Check>
  </Tab>

  <Tab title="Check Account Balance">
    **Verify balance display:**
    
    ```typescript
    // This code is already included in your template
    import { useBalance } from 'wagmi';
    
    const { data: balance } = useBalance({
      address: userAddress,
    });
    ```
    
    <Check>
    Your SEI balance should appear in the interface after wallet connection.
    </Check>
  </Tab>

  <Tab title="Test a Transaction">
    **Try a simple transaction:**
    
    1. Ensure you have some SEI in your wallet (get testnet tokens from [faucet](https://faucet.sei-apis.com))
    2. Use the example transfer function in your template
    3. Send a small amount to another address
    
    <Check>
    Transaction should complete in 1-2 seconds with a transaction hash displayed.
    </Check>
  </Tab>
</Tabs>

## Common patterns

<AccordionGroup>
  <Accordion title="Token transfers" icon="arrow-right-arrow-left">
    **Send SEI or custom tokens:**
    
    ```typescript
    import { useSendTransaction } from 'wagmi';
    import { parseEther } from 'viem';
    
    const { sendTransaction } = useSendTransaction();
    
    // Send 1 SEI
    sendTransaction({
      to: '0x...',
      value: parseEther('1')
    });
    ```
  </Accordion>

  <Accordion title="Smart contract calls" icon="file-contract">
    **Interact with deployed contracts:**
    
    ```typescript
    import { useWriteContract } from 'wagmi';
    
    const { writeContract } = useWriteContract();
    
    // Call a contract function
    writeContract({
      address: '0x...',
      abi: contractAbi,
      functionName: 'transfer',
      args: [recipientAddress, amount]
    });
    ```
  </Accordion>

  <Accordion title="Query blockchain data" icon="magnifying-glass">
    **Read contract state and blockchain data:**
    
    ```typescript
    import { useReadContract } from 'wagmi';
    
    const { data } = useReadContract({
      address: '0x...',
      abi: contractAbi,
      functionName: 'balanceOf',
      args: [userAddress]
    });
    ```
  </Accordion>
</AccordionGroup>
