## Overview

The `@sei-js/create-sei` package is a command-line interface (CLI) tool that helps you quickly scaffold Sei applications. It provides various templates for both EVM and Cosmos development, supporting popular frameworks like Next.js and Vite.js.

## Installation

You don't need to install this package globally. Use it directly with npx:

<CodeGroup>

```bash npx
npx @sei-js/create-sei app
```

```bash yarn
yarn create @sei-js/create-sei app
```

```bash pnpm
pnpm create @sei-js/create-sei app
```

</CodeGroup>

## Quick Start

### Interactive Setup

Run the CLI tool and follow the interactive prompts:

```bash
npx @sei-js/create-sei app
```

The wizard will ask you to choose:

1. **Project Name**: Your application name
2. **Framework**: Next.js or Vite.js
3. **Ecosystem**: EVM or Cosmos
4. **Wallet Library**: Wagmi/Viem or Ethers.js (for EVM)
5. **Styling**: CSS framework preferences
6. **Additional Features**: TypeScript, ESLint, etc.

### Direct Template Selection

You can also specify options directly:

<CodeGroup>

```bash Next.js + EVM + Wagmi
npx @sei-js/create-sei app my-sei-app --template nextjs-evm-wagmi
```

```bash Vite + EVM + Ethers
npx @sei-js/create-sei app my-sei-app --template vite-evm-ethers
```

```bash Next.js + Cosmos
npx @sei-js/create-sei app my-sei-app --template nextjs-cosmos
```

</CodeGroup>

## Template Features

### Common Features

All templates include:

- **TypeScript**: Full type safety out of the box
- **ESLint & Prettier**: Code formatting and linting
- **Tailwind CSS**: Utility-first CSS framework
- **Environment Variables**: Pre-configured for different networks
- **Wallet Connection**: Ready-to-use wallet integration
- **Network Switching**: Support for mainnet, testnet, and devnet
- **Error Handling**: Comprehensive error boundaries and handling
- **Loading States**: Proper loading and pending states
- **Responsive Design**: Mobile-first responsive layouts

### EVM-Specific Features

- **Wagmi Integration**: React hooks for Ethereum
- **Viem Support**: Type-safe, lightweight Ethereum library
- **Ethers.js Option**: Popular Ethereum library support
- **Smart Contract Interaction**: Pre-built contract interaction patterns
- **Token Operations**: Send, receive, and manage tokens
- **Transaction Handling**: Comprehensive transaction management

### Cosmos-Specific Features

- **Cosmos Kit**: Multi-wallet support for Cosmos ecosystem
- **CosmWasm Integration**: Smart contract interaction utilities
- **IBC Support**: Inter-blockchain communication helpers
- **Staking Operations**: Delegation and reward management
- **Governance**: Proposal voting and participation tools

## Customization

### Adding Custom Components

After scaffolding, you can add custom components:

<CodeGroup>

```tsx Custom Token Transfer (EVM)
// src/components/TokenTransfer.tsx
import { useAccount, useWriteContract } from 'wagmi';
import { parseEther } from 'viem';

export function TokenTransfer() {
  const { address } = useAccount();
  const { writeContract } = useWriteContract();

  const handleTransfer = () => {
    writeContract({
      address: '0x...',
      abi: tokenAbi,
      functionName: 'transfer',
      args: [recipientAddress, parseEther('1.0')]
    });
  };

  return (
    <button onClick={handleTransfer}>
      Transfer Token
    </button>
  );
}
```

```tsx Custom Staking Component (Cosmos)
// src/components/StakingDashboard.tsx
import { useChain } from '@cosmos-kit/react';
import { useStaking } from '../hooks/useStaking';

export function StakingDashboard() {
  const { address } = useChain('sei');
  const { delegations, rewards } = useStaking(address);

  return (
    <div className="space-y-4">
      <h2>Staking Dashboard</h2>
      <div>Total Delegated: {delegations.total}</div>
      <div>Pending Rewards: {rewards.total}</div>
    </div>
  );
}
```

</CodeGroup>

## Development Workflow

### Getting Started

After creating your project:

<CodeGroup>

```bash Install Dependencies
cd my-sei-app
npm install
```

```bash Start Development Server
npm run dev
```

```bash Build for Production
npm run build
```

</CodeGroup>

### Available Scripts

All templates include these npm scripts:

- `dev`: Start development server
- `build`: Build for production
- `start`: Start production server
- `lint`: Run ESLint
- `lint:fix`: Fix ESLint errors
- `type-check`: Run TypeScript compiler check

## Troubleshooting

<AccordionGroup>
  <Accordion title="Installation Issues">
    **Problem**: CLI fails to create project
    
    **Solutions**:
    - Ensure Node.js version ≥16
    - Clear npm/yarn cache
    - Check network connectivity
    - Try with different package manager
  </Accordion>

  <Accordion title="Template Issues">
    **Problem**: Template doesn't work as expected
    
    **Solutions**:
    - Update to latest @sei-js packages
    - Check environment variable configuration
    - Verify wallet extension is installed
    - Review console for error messages
  </Accordion>

  <Accordion title="Build Issues">
    **Problem**: Build fails in production
    
    **Solutions**:
    - Check TypeScript errors
    - Verify all dependencies are installed
    - Review environment variable setup
    - Check for missing public assets
  </Accordion>
</AccordionGroup>
